# Phase 22 — Streaming `<main>` landmark restructure (Path a)

> Detailed brief for the autonomous loop. Generated by
> `/ship-a-phase` 2026-05-10 from the PHASE_CANDIDATES expand
> pass-3 row (score 5.5) + the build plan one-liner. Promoted
> via `/oversight` 2026-05-10; Path (a) — per-route `<main>`
> ownership — locked at promotion time.

## Outcome

Every dynamic-data page (`/`, `/trends`, `/trends/tracker`,
`/group-buys`, `/sources`, `/tag/<slug>`, `/article/<slug>`,
`/part/<kind>`, `/part/<kind>/<slug>`, etc.) renders its real
content INSIDE the `<main>` landmark in the SSR'd HTML — not as
a `$RC`-reparented sibling that lands inside `<main>` only after
hydration. Screen-reader users navigating the SSR'd a11y tree
land on actual page content; JS-disabled users (rare, but the
WCAG conformance bar) see a navigable landmark structure.

## Why

Critique pass 7 [HIGH] #22 (filed 2026-05-10) documented that
on every dynamic-data page, `<main>` only contains the
`loading.tsx` placeholder shell pre-hydration; the real content
arrives as a `<template>` block elsewhere in the document and
gets reparented client-side via `$RC` scripts. The pattern
correlates 1:1 with `loading.tsx` presence per route. Static
pages (/about, 404) render content correctly inside `<main>`
because they have no Suspense boundary.

Pass 8 (commit 3955686) re-confirmed the symptom on 6 of 6
visited routes. The row was reclassified `[x]` at /oversight
2026-05-10 with the promotion pointer to this phase.

## The fix shape

**Path (a)** — locked at /oversight time. Move `<main>` from
`apps/web/src/app/layout.tsx` to each route's `page.tsx`. Each
`loading.tsx` also wraps its skeleton in `<main>`. Each
`not-found.tsx` and `error.tsx` does the same. The layout
becomes Header + `{children}` + Footer; each child segment owns
its own `<main>` element.

**Why Path (a) over Path (b)** (drop loading.tsx files): Path
(b) would lose the per-route loading-skeleton UX (perceived
speed win that Next 15 streaming gives for free) without buying
anything Path (a) doesn't already give. Path (a) is the
standard Next 15 + App Router pattern for routes that own their
landmark.

## Render contract per page

Each page/loading/not-found/error component now wraps its
returned tree in:

```tsx
<main className="flex-1">
  {/* JsonLd, Container header, Container body, etc. */}
</main>
```

The `flex-1` class preserves the layout's flex-grow behavior so
`<Footer>` stays at viewport bottom on short pages. Previously
the layout's `<main className="flex-1">` provided this; now each
page does. The contract is enforced by the e2e gate: every
canonical URL renders exactly one `<main>` element.

## Layout change

```tsx
// apps/web/src/app/layout.tsx
// before:
<body ...>
  <GoogleTagManager />
  <Header />
  <main className="flex-1">{children}</main>
  <Footer />
</body>

// after:
<body ...>
  <GoogleTagManager />
  <Header />
  {children}
  <Footer />
</body>
```

The layout no longer claims the `<main>` landmark. Each route
segment (page/loading/not-found/error) is responsible for
emitting exactly one `<main>` element when it renders.

## E2e gate

Update `apps/e2e/tests/smoke.spec.ts` (or add a dedicated
landmark spec) to assert `await expect(page.locator('main')).toHaveCount(1)`
on every HTML canonical URL. The new assertion fails fast if a
page forgets its `<main>` wrapper or accidentally double-wraps.

## Decisions to lock at ship time

1. **JsonLd inside or outside `<main>`?** JSON-LD `<script>`
   tags work anywhere in the body. Keeping them inside the
   `<main>` wrapper minimizes diff — every page that returned
   `<><JsonLd /><Container /></>` becomes
   `<main><JsonLd /><Container /></main>`. The HTML5 spec
   permits `<script>` inside `<main>`. Decision: inside.

2. **Should error/not-found pages also use `<main>`?** Yes —
   they replace the page content, so they must own the landmark.
   A page where `notFound()` fires renders the not-found.tsx;
   that file needs its own `<main>` for the gate to pass.

3. **`flex-1` preserved per page** — every wrapper carries
   `className="flex-1"` (or equivalent flex-grow utility) so
   the body's flex layout still pushes Footer to viewport
   bottom.

## Verify gate

`pnpm verify` must pass:
- typecheck: every page returns valid JSX after the wrap.
- unit: existing 255 pass; no new tests required (the e2e is
  the regression guard for landmark structure).
- data:validate: no schema changes.
- build: every static page generates correctly.
- size: bundle stays under 200 KB on `/`. The wrap is
  HTML-only; no JS weight delta.
- e2e: existing 408 pass + the new "exactly one `<main>` per
  HTML URL" assertion across 70+ canonical URLs.
